## Phase 2: Apply Critique Fixes

For each critical issue:

**Issue: "Misaligned with user context"**
→ Replace misaligned tasks with context-relevant ones

**Issue: "Low strategic value (avg < 7.0)"**
→ Remove low-priority tasks, promote high-impact work

**Issue: "Vague/unactionable tasks"**
→ Add specifics: file paths, technical approach, success criteria

**Issue: "No quick wins"**
→ Add 1 task under 1 hour for momentum

**Issue: "Wrong bottleneck focus"**
→ Refocus on actual constraint (demo quality, user retention, etc.)

**Issue: "Major blockers"**
→ Reorder tasks or split to remove dependencies

---

## Phase 3: Re-Prioritize Using Opus Thinking

For each task, evaluate:

**1. Systems Thinking:**
- Does this address root cause?
- Will this make 10 other things easier?
- Does this create reusable assets?

**2. Strategic Optionality:**
- Does this unlock multiple future paths?
- Does it increase flexibility?
- Can we learn before committing?

**3. Information Value:**
- Will this reveal critical unknowns?
- Does it de-risk assumptions?
- Is this a learning investment?

**4. Compounding Returns:**
- Will this accelerate future work?
- Is this foundational?
- Does value compound over time?

**5. Context Fit:**
- Does this match current stage/constraint?
- Is this the right work for NOW?

**6. Reversibility:**
- Can we undo if wrong? (Type B = move fast)
- Or is this irreversible? (Type A = high scrutiny)

**7. Bottleneck Focus:**
- What's the current constraint?
- Does this address the real bottleneck?

**8. Opportunity Cost:**
- Is this the best use of time?
- What are we NOT doing if we do this?

**Re-score based on above:**
````
Task Priority = 
  (Impact × 0.40) + 
  (Effort_inverse × 0.30) + 
  (Risk_inverse × 0.20) + 
  (Dependencies_inverse × 0.10) +
  Opus_bonuses
````

**Opus Bonuses:**
- High leverage: +1.5
- Strong optionality: +1.0
- Compounding returns: +1.0
- Perfect context fit: +1.5
- Addresses bottleneck: +1.0
