## Exploration summary

**Document:** `.cursor/explore/investor-grade-demo-exploration.md`

### Core technical challenge

Making welding sensor data understandable for non-technical investors through a guided narrative and a zero-backend path from the tech demo to the team management story.

### Approach

1. **Guided tour:** Use a custom overlay instead of a library:
   - Config-driven steps (2–4)
   - Skip always available
   - Optional scrub to specific timestamps (e.g. novice spike)
   - `z-[200]` and `isolate` so it sits above WebGL

2. **Browser-only team path:** Add `/demo/team`:
   - Team dashboard with 2 welder cards (Mike Chen, Expert Benchmark)
   - Welder report at `/demo/team/[welderId]`
   - `getDemoTeamData()` + `createMockScore()` from new `lib/seagull-demo-data.ts`
   - Reuse HeatMap, FeedbackPanel, LineChart, `generateAIFeedback`
   - No `fetchSession` / `fetchScore`

3. **CTA:** “See Team Management” on the demo → `/demo/team`

4. **AppNav:** Add “Team” link → `/seagull` for app users

5. **Preset scenario:** Use `timestamp_ms: 2400` for the novice spike (sine peaks at ~2.4s in `demo-data.ts`)

### Findings

- **Mock SessionScore:** Reuse pattern from `ai-feedback.test.ts` (`mockScore`) for expert (94) and novice (42).
- **`useFrameData`:** Works with in-memory frames from `demo-data.ts`; no refactor needed.
- **Reuse:** Seagull’s HeatMap, FeedbackPanel, LineChart, and `generateAIFeedback` are usable as-is; only the data source changes.
