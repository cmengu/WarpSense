## Quick Reference: Anti-Patterns

### Authentication
```python
# ❌ NEVER: Skip token expiry validation
payload = jwt.decode(token, SECRET_KEY)

# ✅ ALWAYS: Explicitly validate expiry
payload = jwt.decode(token, SECRET_KEY, verify_exp=True)
```

### Database
```sql
-- ❌ NEVER: Add non-nullable to existing tables
ALTER TABLE users ADD COLUMN field TEXT NOT NULL;

-- ✅ ALWAYS: Add nullable first, backfill later
ALTER TABLE users ADD COLUMN field TEXT DEFAULT '';
```

### External APIs
```javascript
// ❌ NEVER: No retry logic
const result = await externalAPI.call(data);

// ✅ ALWAYS: Exponential backoff
const result = await retryWithBackoff(() => externalAPI.call(data));
```

### Frontend State
```typescript
// ❌ NEVER: Mutate state directly
state.items.push(newItem);

// ✅ ALWAYS: Create new reference
setState({ ...state, items: [...state.items, newItem] });
```

## AI Prompting Patterns

### Auth Implementation
```
Implement JWT validation that:
1. Verifies expiry explicitly (see LEARNING_LOG.md entry [date])
2. Validates signature with SECRET_KEY
3. Extracts user_id from token, never request
4. Handles ExpiredSignatureError, InvalidTokenError
5. Logs failures with context (user_id, IP, timestamp)
```

### Database Migrations
```
Create migration for [field] on [table]:
1. Handle existing data (assume 500k+ rows)
2. Use nullable + default value approach
3. Include rollback SQL in comments
4. Test on production-sized dataset
```

### API Integration
```
Integrate [API] with:
1. Exponential backoff (tenacity library)
2. Circuit breaker (pybreaker)
3. Timeout (5s default, 30s max)
4. Response caching where appropriate
5. Quota monitoring with alerts
```

# Task:

Beautiful PDF Report
Philosophy
Don't screenshot the UI. Build the PDF as a first-class layout. @react-pdf/renderer lets you define PDF-native components — no DOM, no canvas capture hacks, no puppeteer. It renders on the server in a Next.js API route.
Step 1: Install
bashnpm install @react-pdf/renderer
npm install -D @types/react-pdf
Step 2: PDF Layout Component (components/pdf/WelderReportPDF.tsx)

Step 3: Chart → PNG (the tricky bit)
@react-pdf/renderer can't render Recharts. You need to pre-render the trend chart to a base64 PNG. Two options:
Option A (fast, good enough): Use html-to-image on the client to capture the existing LineChart div into a PNG, pass it as base64 to the PDF route. Ship this first.
Option B (clean, proper): Build a <canvas> chart in the API route using chartjs-node-canvas (server-side Chart.js). More work, no DOM dependcy.
Start with Option A.

Step 4: PDF API Route
typescript// app/api/welder-report-pdf/route.ts
import { renderToBuffer } from '@react-pdf/renderer';
import { WelderReportPDF } from '@/components/pdf/WelderReportPDF';

export async function POST(req: Request) {
  const { welder, score, feedback, chartDataUrl } = await req.json();
  
  const buffer = await renderToBuffer(
    <WelderReportPDF welder={welder} score={score} 
                     feedback={feedback} chartDataUrl={chartDataUrl} />
  );
  
  return new Response(buffer, {
    headers: {
      'Content-Type': 'application/pdf',
      'Content-Disposition': `attachment; filename="${welder.name}-report.pdf"`,
    },
  });
}
Step 5: Replace the stub button
typescriptasync function handleDownloadPDF() {
  const chartEl = document.getElementById('trend-chart');
  const chartDataUrl = chartEl ? await toPng(chartEl) : null;
  
  const res = await fetch('/api/welder-report-pdf', {
    method: 'POST',
    body: JSON.stringify({ welder, score, feedback, chartDataUrl }),
  });
  
  const blob = await res.blob();
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `${welder.name}-warp-report.pdf`;
  a.click();
}
