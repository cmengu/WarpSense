## What to Check (Exhaustive)

### 1. Logging & Debugging (CRITICAL for production)
- ‚ĚĆ **No console.log, console.error, console.warn** anywhere
- ‚ĚĆ **No debugger statements**
- ‚ĚĆ **No alert() or confirm()** (except intentional UI)
- ‚úÖ Use proper logger with context (winston, pino, or project standard)
- ‚úÖ Log levels appropriate (error, warn, info, debug)
- ‚úÖ Sensitive data not logged (passwords, tokens, PII)

**Check each file for:**
```typescript
// BAD
console.log('user data:', user);
debugger;

// GOOD  
logger.info('User authenticated', { userId: user.id, timestamp: Date.now() });
```

### 2. Error Handling (CRITICAL)
- ‚úÖ **All async functions** have try-catch blocks
- ‚úÖ **All API calls** handle network failures
- ‚úÖ **All file operations** handle missing files
- ‚úÖ **All JSON.parse()** wrapped in try-catch
- ‚úÖ Error messages are helpful (not "Error" or "Failed")
- ‚úÖ Errors logged with full context
- ‚úÖ User-facing errors are sanitized (no stack traces to users)
- ‚úÖ Error boundaries present (if React)

**Check for:**
```typescript
// BAD - no error handling
const data = await fetch('/api/data').then(r => r.json());

// GOOD
try {
  const response = await fetch('/api/data');
  if (!response.ok) throw new Error(`HTTP ${response.status}`);
  const data = await response.json();
} catch (error) {
  logger.error('Failed to fetch data', { error, url: '/api/data' });
  throw new Error('Unable to load data. Please try again.');
}
```

### 3. TypeScript Strict Mode (HIGH)
- ‚ĚĆ **No `any` types** anywhere (use `unknown` if truly needed)
- ‚ĚĆ **No `@ts-ignore` or `@ts-expect-error`** without explanation
- ‚ĚĆ **No implicit any** (check function params, returns)
- ‚úÖ All props/interfaces defined
- ‚úÖ Return types explicit on functions
- ‚úÖ Strict null checks respected (no `!` operator without reason)
- ‚úÖ Enums or unions instead of magic strings
- ‚úÖ Generic types properly constrained

**Flag violations like:**
```typescript
// BAD
function process(data: any) { ... }
const result = data as MyType; // @ts-ignore
let value;  // implicit any

// GOOD
function process(data: ProcessInput): ProcessOutput { ... }
const result = isMyType(data) ? data : null;
let value: string | null = null;
```

### 4. Production Readiness (HIGH)
- ‚ĚĆ **No TODOs, FIXMEs, or XXX comments**
- ‚ĚĆ **No hardcoded secrets, API keys, passwords**
- ‚ĚĆ **No hardcoded URLs** (use env vars)
- ‚ĚĆ **No test/debug code** left in production paths
- ‚ĚĆ **No commented-out code** blocks
- ‚úÖ Environment variables properly used
- ‚úÖ Feature flags if needed for rollout
- ‚úÖ Build passes without warnings

### 5. React/Hooks (if applicable) (HIGH)
- ‚úÖ **All useEffect hooks have cleanup** if needed
- ‚úÖ **Dependency arrays complete** (no eslint-disable)
- ‚úÖ **No infinite loops** (effect triggering itself)
- ‚úÖ **No missing dependencies** in useCallback/useMemo
- ‚úÖ Keys on list items (and not using index)
- ‚úÖ State updates are immutable
- ‚úÖ No direct DOM manipulation (use refs properly)
- ‚úÖ Conditional rendering doesn't break hooks order

**Common issues:**
```typescript
// BAD

  fetchData(); // missing cleanup, missing deps

// GOOD

  const controller = new AbortController();
  fetchData(controller.signal);
  return () => controller.abort(); // cleanup

```

### 6. Performance (MEDIUM)
- ‚úÖ **No unnecessary re-renders** (memo, useMemo, useCallback used wisely)
- ‚úÖ **Expensive calculations memoized** (useMemo)
- ‚úÖ **Large lists virtualized** (if >100 items)
- ‚úÖ **Images optimized** (next/image or similar)
- ‚úÖ **Bundle size impact** considered
- ‚úÖ **No N+1 queries** (if database access)
- ‚úÖ **Lazy loading** for routes/components (if applicable)
- ‚úÖ **Debouncing/throttling** for frequent events

**Check for:**
```typescript

// GOOD

### 7. Security (CRITICAL)
- ‚úÖ **Auth checked** before sensitive operations
- ‚úÖ **Inputs validated** (type, format, length, range)
- ‚úÖ **SQL injection prevented** (parameterized queries)
- ‚úÖ **XSS prevented** (no dangerouslySetInnerHTML without sanitization)

- ‚úÖ **RLS policies** in place (if using Supabase/Postgres)

- ‚úÖ **Sensitive data not in client** (tokens, secrets)

**Flag issues like:**

```
