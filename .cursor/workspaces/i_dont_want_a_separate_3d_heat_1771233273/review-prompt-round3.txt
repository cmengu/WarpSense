## Common Fix Verification Failures

Watch for these signs of incomplete fixes:

### 🚩 Band-Aid Fix
```typescript
// Bad: Just suppressing the error
try { riskyOperation(); } catch { } // ❌ Silent failure

// Good: Proper error handling
try { 
  riskyOperation(); 
} catch (error) {
  logger.error('Operation failed', { error });
  setUserError('Unable to complete action');
}
```

### 🚩 Incomplete Fix
```typescript
// Bad: Fixed in one place, not others
// File A: Fixed ✅
const query = 'SELECT * FROM users WHERE id = $1';

// File B: Still vulnerable ❌  
const query = `SELECT * FROM users WHERE email = '${email}'`;
```

### 🚩 Introduces New Problem
```typescript
// Bad: Fixed SQL injection, introduced memory leak
async function getUser(id) {
  const client = await pool.connect();
  return await client.query('SELECT * FROM users WHERE id = $1', [id]);
  // ❌ Client never released
}
```
